# ManagerReady AI Growth Plan

Date: 2026-05-08

## Goal

Build low-cost acquisition for ManagerReady AI through search, GEO, examples, templates, and direct generator conversion.

The product should not rely on daily manual posting. The first growth system should create durable pages that answer high-intent work-update queries and send users into the generator.

## Positioning

ManagerReady AI helps non-native English professionals turn rough multilingual work notes into polished manager-ready English updates.

Primary audiences:

1. Non-native English software engineers
2. Non-native English product managers and operators
3. Remote team members
4. Freelancers, agencies, and client-facing delivery teams
5. Professionals preparing performance reviews or promotion summaries

## Acquisition Strategy

The main path is:

Search or AI answer surface -> template/example/answer page -> scenario-prefilled generator -> generation -> copy/refine -> repeat use or paid offer later.

This favors content that already matches user intent:

- Weekly update generator
- Manager update generator
- Client update email template
- Engineering weekly report example
- ChatGPT prompt for weekly status update
- English work update for non-native speakers
- Performance review summary example

## Page Requirements

Every growth page should include:

- One clear search-intent title
- Short answer or explanation near the top
- Before/after example when relevant
- Copyable template when relevant
- Link back to the generator with scenario and UTM parameters
- Links to related templates, examples, guides, or use cases
- Metadata and canonical URL
- Structured data when useful
- Inclusion in sitemap and llms.txt

## 30-Day Execution Plan

### Week 1: Build Search Assets

- Add template detail pages
- Add example detail pages
- Add GEO answer pages
- Update sitemap and llms.txt
- Link all growth pages back to the generator

### Week 2: Improve Conversion

- Add "Use this template" and "Use this example" links
- Pass scenario and UTM values into generator links
- Track source page in PostHog events
- Add related content blocks between pages

### Week 3: Indexing and Distribution Prep

- Prepare Search Console and Bing Webmaster submission checklist
- Add Open Graph metadata
- Prepare launch copy for Product Hunt, Indie Hackers, Reddit, and LinkedIn
- Create a short product description and FAQ for directory submissions
- Use `docs/distribution-plan.md` as the 14-day active distribution sprint
- Use `docs/social-posts.md`, `docs/reddit-posts.md`, and `docs/launch-kit.md` as the first posting asset library
- Track directory submissions in `docs/directory-submissions.csv`
- Record weekly channel learnings in `docs/distribution-results.md`

### Week 4: Measure and Expand

- Review pages that get impressions or generator clicks
- Expand pages around the best-performing scenarios
- Rewrite weak pages with stronger examples
- Decide whether to add email capture or a paid template pack

## What Codex Can Implement Directly
